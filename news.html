Open source software for your weather station

News

02/05/15 Version 3.1.0 released
Lots of little things. Support for AcuRite weather stations. SFTP sessions now possible. Minor bug fixes. The rainRate wars finally end.
12/07/14 Version 3.0.1 released
Minor bugfix to the installer and improved log message for missing data bindings.
12/05/14 Version 3.0.0 released
Big release with lots of new features. Overall goal was to continue to make it easier for extension writers. Several new tags, such as $hour, $hours_ago, $days_ago and $yesterday, have been added. Be sure and read the Upgrading Guide.
11/10/14 Version 2.7.0 released
Added EEPROM calibration and configuration options for Vantage stations. Better behavior and reporting for rsync. Added catchup for the ws28xx driver. Support for RainWise CC3000 in the debian, redhat, and suse packages. Enabled localization of weewx and server uptimes. Numerous minor bugfixes.
06/16/14 Version 2.6.4 released
Minor bugfixes. New device drivers for PeetBros Ultimeter, ADS WS1, RainWise Mark III stations and CC3000 data logger. Fix WU and CWOP post problems for WMR100 stations.
04/10/14 Version 2.6.3 released
Accept UTF8 characters in plots. Improved WMR100 driver. CWOP postings are more robust. Battery stats for vantage are now reported in the archive record. Provide better pairing feedback in the ws28xx driver. Other minor bugfixes.
02/16/14 Version 2.6.2 released
Fixed bugs in the new RESTful code, including a bug when rain data is missing and RapidFire identifiers. Fixed bug in WMR200 driver when outTemp is missing. Added timeout to sqlite connections.
02/08/14 Version 2.6.1 released
Fixed a bug in StdQC that could cause weewx to terminate when bad data were encountered.
02/08/14 Version 2.6.0 released
This release was mostly focused on making it easier to write extensions for weewx. In particular, there is a new RESTful architecture that simplifies writing these services. It's now easier to slip a new weewx service into the right place in the list of services to be run. A model has also been defined for structuring and packaging extensions, along with an installer. Lots of other small enhancements and bug fixes.
12/30/13 Version 2.5.1 released
Added drivers for LaCross WS2300 and Hideki TE923 stations. Small bug fix to search list extension and numerous small bugfixes to the forecasting module.
10/29/13 Version 2.5.0 released
Many new, small features, including easier search list extensions, a forecasting module, easier localization, and some new statistical types. You can also have your installation "phone home" and post your station location on a map. See the change log for details.
08/03/13 Version 2.4.0 released
Various small bug fixes and modest enhancements. See the change log for details.
04/10/13 Version 2.3.0 released
Experimental support for the WMR200 and WS28xx. Various bug fixes.
02/14/13 Version 2.2.0 released
Weewx can now be installed using Debian (DEB) or Redhat (RPM) packages, we well as with the old "setup.py" method. New plot options for adding day/night bands and other features.
01/02/13 Version 2.1.0 released
Includes support for the Oregon Scientific 918/968 weather stations and for the Fine Offset weather stations. Added file uploading using rsynch (as well as FTP). Added $trend tag.
11/4/12 Version 2.0.0 released
Includes support for the Oregon Scientific WMR-USB series, as well as support for the MySQL database.

Allows more hardware configuration options to be set in the Davis EEPROM.
10/06/11 Version 1.11.0 released
Offers support for the Davis WeatherLinkIP. Posts UV and radiation data to the WeatherUnderground (if available).
03/29/11 Version 1.10.0 released
Offers extensive almanac information (such as azimuth, elevation of various planetary bodies). Includes mobile formatted page (thanks, Vince!). Tags can now override unit types. Users can reconfigure the main database, adding and dropping data types. Users can also add custom data types.
07/17/10 Ported to Mac OS X
User Harry Felder has ported weewx to the Mac OS X operating system. No source code changes, but he has provided a ZIP file that includes a persistent daemon that does the start up and shut down of weewx, including after I/O errors. See the Users' Group file section.
07/04/10 Version 1.9.0 released
Offers a rudimentary quality control: observation values are checked to make sure they lie within a specified range.
04/28/10 Version 1.8.0 released
Now offers correction expressions for applying calibration corrections to your instrument. Added CWOP support. Now saves console battery voltage and transmit battery state to the archive.
04/15/10 Version 1.7.0 released
This is a big release with many new features, including support for skins. A skin is a collection of the templates, CSS style sheets, GIFs, and other items necessary to run a report. This makes it possible to run multiple reports against the same skin and put the results in different subdirectories. For example, US and Metric versions of a report could be produced. Many other new features including RSS feeds, support for PWSWeather, support for UTF-8 encoding, many new formatting options, etc. See the change log.
03/07/10 Version 1.5.0 released
This new version supports metric units in the presentation layer. It is possible to mix and match which units are used for which variables. For example, US Customary could be used for all variables except pressure, which could use millibars. Other combinations are possible.
02/22/10 Version 1.4.0 released
Changed to a late binding architecture for running statistical summaries. Now the SQL statements are not run until template evaluation. This reduced the memory requirements by one half, made execution slightly faster (at least on my server), and simplified the statistics code by 150 lines. Added the feature of being able to reread the configuration file without stopping the process.
12/22/09 Version 1.3.0 released
The internal architecture of weewx was changed. It now consists of an engine that loads and manages services. New services can easily be created. This makes weewx much easier to customize and extend for specialized applications. Posting on the Weather Underground is now just another service. Includes a sample new "Alarm" service, where an email is sent if an arbitrary expression evaluates true. New document on customizing weewx.

Mostly, this is just like any other Ubuntu install. Most of the effort was in getting the Plug ready. Here are my install notes.
